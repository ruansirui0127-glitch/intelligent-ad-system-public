# 产品愿景

## 一句话定位

「智能投放系统」是一套面向广告增长团队的智能投放操作系统，帮助团队把市场洞察、素材生产、广告投放、销售承接、试听转化、成交复盘和策略学习连接成一个可解释、可审批、可追责、可迭代的增长闭环。

系统不是单一广告看板，也不是纯自动化脚本。它的价值在于把分散在平台后台、素材文档、销售记录、复盘会议和团队经验里的判断，沉淀成可被系统执行、审计和学习的工作流。

## 完整增长链路

系统覆盖两条互相咬合的链路。

第一条是投放决策链路：

`市场信号 → 投放假设 → 素材 Brief → 素材生产 → 素材库沉淀 → 广告创建草稿 → 受控投放 → 数据同步 → 异常诊断 → 证据解释 → 优化建议 → ActionIntent 审批 → 执行记录 → 动作复盘 → 策略/规则/素材回流 → 下一轮测试`

第二条是业务转化链路：

`曝光 → 点击 → 表单/H5/WhatsApp → 线索 → 预约试听 → 到课 → 成交 → ROI/LTV → 复购或转介绍`

系统要做的是把这两条链路接起来：投放动作不能只用平台成本评价，还要用线索质量、约课、到课、成交和 ROI/LTV 校准；素材生产不能只看创意喜好，还要看投放表现和后链路质量；策略复盘不能只停留在会议结论，而要回流到规则、素材标签、预算策略和下一轮测试矩阵。

## 业务背景

当前业务面向海外华人家庭，为 4-12 岁儿童提供数学思维在线直播课。第一阶段以 Meta 平台作为广告数据和投放诊断主线，核心商业链路从广告触达开始，经过留资、销售承接、试听预约、到课体验，最终转化为正式付费。

不同地区处于不同市场阶段：成熟市场需要识别衰退、素材疲劳和效率下降；增长市场需要承受合理测试成本并找到扩量窗口；核心基本盘需要持续比较渠道质量、地区表现和 ROI。

因此，系统不能只追求最低 CPL。它必须逐步建立从前端广告效率到后链路商业质量的统一判断。

## 核心问题

当前增长团队面对的不是一个单点工具问题，而是链路断裂问题。

- 市场信号、竞品变化和家长焦虑没有稳定转成投放假设。
- 素材 brief、素材版本、素材标签和素材表现没有形成可学习资产。
- 广告账户、计划、单元、创意、关键词、人群和地区数据分散在平台后台。
- 日常诊断依赖人工经验，不同人对扩量、止损、疲劳、异常的判断不一致。
- 优化动作缺少审批、日志、冷却时间和效果追踪。
- 销售承接、约课、到课、成交和 ROI 没有稳定回流到投放判断。
- 复盘结论没有结构化进入下一轮规则、预算、素材和测试计划。

## 产品原则

1. 先解释，再建议，再审批。
   每个建议必须带触发指标、历史基线、对比对象、样本量、业务影响、风险和建议动作。

2. 用后链路质量校准平台指标。
   CPL 只是前端效率指标，系统要逐步接入有效线索率、接通率、约课率、到课率、成交成本、ROI 和 LTV。

3. 把素材变成增长资产。
   素材要绑定地区、人群、痛点、卖点、渠道、版本、创意形式、投放假设和表现数据。

4. 高风险动作必须受控。
   创建广告、调预算、关停、改出价、改归因配置必须有审批、日志、冷却时间和回滚边界。

5. 统一抽象，不抹平差异。
   系统内部用统一对象连接多平台，但保留各平台在目标、字段、审核、归因和动作权限上的差异。

6. 让复盘进入系统记忆。
   每一次操作、采纳、拒绝和效果变化都应成为后续规则、brief、预算模型和 playbook 的候选学习样本。

## V0 要验证什么

V0 先验证「投放优化师工作台 + 受控 agent 建议链路」是否成立。

V0 的最小闭环是：

`数据同步 → 规则命中 → EvidenceObject → ContextPackage → agent 建议 → Recommendation → ActionIntent 草稿 → 审批/拒绝 → ActionAudit → 复盘样本`

V0 必须跑通：

- 用户能看到全量账户层级数据，而不是只看风险卡片。
- 诊断能引用规则版本、历史基线、样本量和证据对象。
- agent 只能基于系统给定的 ContextPackage 输出建议。
- 建议能转成 ActionIntent 草稿。
- ActionIntent 可以审批或拒绝，但 V0 不直接执行真实投放动作。
- 每条建议能追溯到数据来源、时间窗口、指标口径、agent 输入输出和工具日志。

## V0 核心页面

- 今日工作台：风险、机会、待审批、数据质量和 agent 摘要。
- 账户分析与诊断页：账户、计划、单元、创意、素材、关键词/人群的全量浏览和诊断叠加。
- 规则配置中心：诊断规则、阈值、观察窗口、样本量、风险等级、启用状态和版本。
- ActionIntent 审批页：建议动作、证据、风险、参数调整、审批/拒绝和执行状态。
- 素材增长中台：素材库、标签、表现、brief、竞品洞察卡片和生成任务入口。
- 数据与 Harness 监控：同步状态、指标口径、归因质量、ContextPackage、AgentTask 和 ToolCallAudit。

## 用户与成功体验

投放师每天打开系统后，能知道今天最该处理什么、为什么要处理、系统建议做什么、证据是否可信，以及这个动作后续如何复盘。

投放负责人能看到预算效率、渠道质量、地区表现、风险趋势、后链路质量和 ROI/LTV 变化，判断哪里加预算、哪里止损、哪里需要换素材或调整承接。

素材/内容团队能看到哪些卖点、地区、人群、形式和版本有效，基于市场信号和历史表现生成下一批素材 brief。

销售/运营团队能反馈有效线索、接通、约课、到课、成交和无效原因，让系统判断问题发生在流量、素材、承接还是课程转化。

数据/系统管理员维护指标口径、字段映射、同步状态、授权状态、归因质量、规则版本和审计日志。

## 当前不做

- 不做无审批的真实预算、出价、暂停、创建广告动作。
- 不做所有平台的深度接入。
- 不做复杂角色权限。
- 不把单日 CPL 作为唯一优化目标。
- 不让 agent 直接连接数据库或直接调用广告平台执行花钱动作。
