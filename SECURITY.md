# 安全策略

## 仓库公开原则

这个仓库按公开仓库管理。任何提交到 Git 的内容，都默认可能被外部人员、搜索引擎和自动扫描工具看到。

允许公开：

- 通用代码、接口契约、开发模板和架构说明。
- 明确标注为 synthetic、fake、sample 的合成样例。
- `config/examples/` 下的配置模板。
- 不包含内部数据的测试用例。

禁止公开：

- token、refresh token、app secret、Cookie、私钥、证书、恢复码。
- 真实广告账户 ID、真实广告数据、平台导出报表、后台截图。
- 真实客户、学生、家长、供应商、订单、合同、报价、营收、成本、利润数据。
- 运行数据库、日志、浏览器抓包、聊天记录、会议纪要、AI 对话原文。
- 未审核的 Word、PDF、Excel、CSV、ZIP、截图和录屏。

## 私有目录

真实配置和运行态只能放在这些 Git 忽略目录：

```text
config/private/
runtime/private/
secrets/
private/
```

如果需要给别人演示，请使用 `config/examples/` 和 `data/sample/` 中的公开模板。

## 发现泄露怎么办

如果发现真实密钥或业务数据已经提交：

1. 立即停止继续 push。
2. 立刻轮换相关 token 或密钥。
3. 把仓库临时改回 private。
4. 从当前版本删除风险文件。
5. 评估是否需要清洗 Git 历史。

删除当前文件不能自动删除历史提交里的内容；历史清洗需要单独执行并通知所有协作者重新 clone。
